1 Introduction

Ventricular Tachycardia (VT) and Ventricular Fibrillation (VF) are the two most frequent recorded shockable rhythms (i.e., life-threatening ventricular arrhythmias) causing Sudden Cardiac Death (SCD) [1]. The design of Implantable Cardioverter-Defibrillator (ICD) is critical in the treatment of ventricular arrhythmia. Since it can deliver defibrillation for patients experiencing life-threatening ventricular arrhythmias. Meanwhile, deep learning can potentially revolutionized ICD design by extracting features not easily identifiable by or even unknown to human experts. However, while many machine learning competitions exist, they mostly focus on computer vision only and does not utilize hardware platforms to such an extreme limit (tinyML).

Therefore, the goal of this contest is to develop a tool to achieve extremely low power budget, memory footprint, low latency and high accuracy on extremely resource-constraint embedded platforms (flash memory < 128K, power consumption < 1 uW, latency < 5ms, accuracy > 90%). That is, contestants are required to write a program to identify a given IECG and implement it on extremely resource-constraint embedded platforms.


  

2 Problem Formulation and Input Output Format

This section defines the input, output, and the requirement of this contest.

Given an IECG data set containing 2-15 minutes IEGM recordings over 95 subjects with 250 Hz sampling rate, 16-bit resolution and 15-55 Hz bandpass filter. Contestants need to realize the classification of these records on STM32 platform, and minimize the power consumption budget, memory footprint and latency on the premise of ensuring accuracy.

2.1 program requirement
The requested program must be run on a Linux system.The program should accept two arguments, “-input < input_path/name.v >” and “-output < output_path/name.v >”, for specifying the input and output files, respectively.
